# PRD: 테트리스 (가제)

## 가정 사항
- 사용자가 별도 규칙을 지정하지 않아, 공식 Tetris Guideline을 기본값으로 채택함: SRS(Super Rotation System) 회전, 7-bag 랜덤 생성, 홀드 기능, 고스트 피스, 넥스트 큐 5개 표시.
- 타겟 플랫폼은 프로젝트 CLAUDE.md 기준 웹 브라우저(데스크톱 우선, 모바일 터치 대응은 P1).
- 멀티플레이/대전 모드는 요청에 없었으므로 범위에서 제외(향후 확장 가능성으로만 언급).
- 로그인/서버 저장 없이 로컬(브라우저) 최고 점수 저장을 기본으로 가정.

---

## 1. 프로젝트 개요

- **게임 제목(가제)**: 테트리스 (Tetris Clone)
- **장르**: 낙하형 퍼즐 (Falling Block Puzzle)
- **타겟 플랫폼**: 웹 브라우저 (데스크톱 키보드 우선, 모바일 터치 P1)
- **타겟 유저층**: 캐주얼 퍼즐 게임을 즐기는 전 연령대, 특히 빠르게 한 판 즐기고 싶은 유저

**엘리베이터 피치**: 10x20 보드에 떨어지는 7종 테트로미노를 회전·이동시켜 가로줄을 채우면 그 줄이 사라지고 점수를 얻는, 규칙이 단순하지만 숙련도에 따라 무한히 깊어지는 클래식 퍼즐 게임.

**핵심 재미(Core Fun)**
- 빠른 판단과 손놀림으로 블록을 정확히 배치하는 순간의 쾌감
- 여러 줄을 동시에 지우는 "테트리스"(4줄 클리어) 달성 시의 짜릿함
- 레벨이 오를수록 빨라지는 속도에 적응하며 몰입하는 긴장감
- 리플레이성: 매번 다른 블록 순서(7-bag)로 매판이 새로움

**차별점**: 표준 Tetris Guideline을 충실히 따라 기존 테트리스 유저에게 익숙한 조작감을 제공하면서, 웹에서 설치 없이 즉시 플레이 가능.

## 2. 핵심 기능 목록

### 2.1 코어 게임플레이 시스템

| 기능명 | 목적 | 동작 방식 |
|---|---|---|
| 보드 | 게임의 기본 무대 | 가로 10칸 × 세로 20칸(+ 상단 여유 2~4칸 버퍼존). 셀 단위 그리드 상태를 2차원 배열로 관리 |
| 테트로미노 7종 | 게임의 핵심 조작 대상 | I, O, T, S, Z, J, L 블록. 각 블록은 4개의 회전 상태(0°/90°/180°/270°)에 대한 셀 좌표 테이블을 가짐 |
| 낙하(Gravity) | 시간에 따라 블록이 내려오는 긴장감 제공 | 레벨별 낙하 속도 테이블 사용(레벨 1: 1000ms/칸 → 레벨이 오를수록 지수적으로 감소, 최저 낙하 간격 하한 설정) |
| 좌우 이동 / 소프트 드롭 / 하드 드롭 | 플레이어의 직접 조작 | 방향키 좌/우: 1칸 이동(홀드 시 DAS/ARR 적용). 아래키: 소프트 드롭(낙하 속도 가속, 점수 +1/칸). 스페이스: 하드 드롭(즉시 바닥까지, 점수 +2/칸) |
| 회전 (SRS) | 블록 배치의 전략적 깊이 제공 | 상/X키: 시계 방향, Z키: 반시계 방향. SRS 킥 테이블 적용(벽/바닥 근처에서 회전 시 위치 보정 시도) |
| 라인 클리어 | 점수 획득의 핵심 트리거 | 가로 한 줄이 모두 채워지면 해당 줄 제거, 위쪽 줄들을 한 칸씩 아래로 이동. 1~4줄 동시 클리어 판정(4줄 = "테트리스") |
| 게임 오버 판정 | 라운드 종료 조건 | 새 블록이 스폰 위치에 겹쳐서 생성될 수 없을 때(Block Out) 게임 종료 |

### 2.2 보조 시스템

| 기능명 | 목적 | 동작 방식 |
|---|---|---|
| 넥스트 큐 | 다음 블록을 예측해 전략적 배치 가능 | 화면 우측에 다음 등장할 블록 5개를 순서대로 표시 |
| 홀드 | 블록을 보관해뒀다 나중에 사용하는 전략 옵션 | C키: 현재 블록을 홀드 슬롯에 저장하고 대기 중이던 블록(또는 큐의 다음 블록)을 소환. 한 블록 스폰당 1회만 사용 가능(연속 홀드 방지) |
| 고스트 피스 | 하드 드롭 위치를 미리 확인해 실수 방지 | 현재 블록이 바로 아래로 떨어졌을 때 도달할 위치를 반투명 윤곽선으로 보드에 표시 |
| 7-bag 랜덤 생성 | 특정 블록이 연속으로 안 나오거나 너무 안 나오는 편향 방지 | 7종 블록을 한 세트로 묶어 무작위 셔플 후 순서대로 소환, 소진되면 다시 셔플 |
| 점수/레벨/라인 카운트 UI | 플레이어에게 현재 진행 상황 피드백 | 화면 상단/측면에 점수, 현재 레벨, 누적 클리어 라인 수 실시간 표시 |

### 2.3 성장/보상 시스템

| 기능명 | 목적 | 동작 방식 |
|---|---|---|
| 점수 시스템 | 플레이 성과를 정량적으로 보여줌 | 라인 클리어 점수(1줄:100×레벨, 2줄:300×레벨, 3줄:500×레벨, 4줄/테트리스:800×레벨), 소프트/하드 드롭 보너스 포함 |
| 레벨 시스템 | 난이도를 점진적으로 높여 몰입 유지 | 누적 클리어 라인 수가 임계값(예: 10라인)을 넘을 때마다 레벨 +1, 낙하 속도 증가 |
| 최고 점수(로컬) | 재도전 동기 부여 | 브라우저 로컬스토리지에 최고 점수 저장, 게임 오버 화면에 "이번 점수 vs 최고 점수" 표시 |

### 2.4 UI/화면

| 화면 | 구성 요소 |
|---|---|
| 타이틀 화면 | 시작하기 버튼, 조작법 안내, (P1) 설정(키 리매핑, 사운드) |
| 인게임 화면 | 보드, 넥스트 큐, 홀드 슬롯, 점수/레벨/라인 표시, 일시정지 버튼 |
| 일시정지 화면 | 이어하기 / 재시작 / 타이틀로 나가기 |
| 게임 오버 화면 | 최종 점수, 최고 점수 갱신 여부, 재시작 / 타이틀로 나가기 |

## 3. 사용자 시나리오

### 페르소나
- **페르소나 A "캐주얼 민지"**: 퍼즐 게임을 가끔 즐기는 20대. 규칙은 알지만 SRS 킥 같은 세부 규칙은 모름. 짧게 한두 판만 하고 나감.
- **페르소나 B "하드코어 태현"**: 테트리스 대회 영상을 즐겨보는 유저. 정확한 SRS 회전, DAS/ARR 세팅, 7-bag 여부에 민감하게 반응함.

### 시나리오 1: 신규 유저 첫 플레이 (온보딩)
1. 타이틀 화면에서 "시작하기" 클릭
2. 조작법 안내(방향키/스페이스/C/상하좌우 등)가 3초간 오버레이로 표시된 후 자동으로 사라짐
3. 카운트다운(3-2-1) 후 첫 블록 스폰, 게임 시작
4. 첫 라인 클리어 시 짧은 시각/사운드 피드백으로 "성공했다"는 감각을 즉시 전달
5. 실수로 블록을 쌓다가 게임 오버 → 점수와 함께 "다시하기" 버튼이 눈에 띄게 표시되어 재도전 유도

### 시나리오 2: 핵심 루프 반복 플레이
1. 유저가 넥스트 큐를 보고 다음 블록을 미리 계획
2. 고스트 피스를 참고해 하드 드롭으로 빠르게 배치
3. 애매한 블록(예: S/Z)은 홀드에 저장하고 다른 블록을 먼저 처리
4. 여러 줄을 동시에 채워 "테트리스" 클리어 → 강한 시각 이펙트 + 사운드로 보상감 극대화
5. 레벨업 시 낙하 속도가 빨라짐을 체감 → 더 빠른 판단 요구받으며 몰입도 상승
6. 게임 오버 후 최고 점수 갱신 여부 확인 → 즉시 "재시작" 눌러 재도전

### 시나리오 3: 엣지 케이스 - 일시정지 중 창 전환
1. 플레이 중 유저가 다른 브라우저 탭으로 전환(또는 창 포커스 아웃)
2. 게임이 자동으로 일시정지되어 블록이 떨어지지 않음(부정확한 낙하로 인한 억울한 게임 오버 방지)
3. 탭으로 복귀 시 일시정지 화면이 표시되고, 유저가 직접 "이어하기"를 눌러야 재개

### 시나리오 4: 엣지 케이스 - 회전 킥 실패
1. 유저가 벽에 붙은 상태에서 T블록을 회전 시도
2. SRS 킥 테이블에 정의된 보정 위치를 순서대로 시도
3. 모든 보정 위치에서도 겹침이 발생하면 회전을 취소(원래 상태 유지)하고 별도 에러 없이 조용히 무시

## 4. 기술 스택

(day4 프로젝트 CLAUDE.md 기준 스택과 통일)

- **언어**: TypeScript (strict 모드)
- **빌드 도구**: Vite
- **게임 렌더링/루프**: Phaser 3
  - 보드 렌더링, 블록 스프라이트, 애니메이션(라인 클리어 이펙트) 담당
  - `systems/` 레이어에 순수 로직(회전 판정, 라인 클리어 판정, SRS 킥 테이블, 7-bag 생성기)을 분리해 Phaser 없이도 유닛 테스트 가능하게 구성
- **UI(비게임 화면)**: React
  - 타이틀 화면, 일시정지 오버레이, 게임 오버 화면, 설정 화면 등 Phaser 캔버스 위에 얹는 오버레이 UI
- **상태 관리**: Zustand (점수, 레벨, 홀드 슬롯 상태 등 Phaser ↔ React 간 공유 상태)
- **로컬 저장**: 브라우저 `localStorage` (최고 점수, 키 설정)
- **테스트**: Vitest로 `systems/`의 회전 판정, 라인 클리어, 7-bag 로직, 점수 계산 단위 테스트 작성
- **입력 처리**: 키보드 이벤트 기반, DAS(Delayed Auto Shift)/ARR(Auto Repeat Rate) 타이머는 별도 입력 시스템 모듈로 구현(향후 리매핑/모바일 터치 확장을 고려한 추상화)

## 5. 개발 우선순위

### MVP 범위 (P0) — "플레이 가능한 최소 단위"
- 10x20 보드, 7종 테트로미노 스폰
- 좌우 이동, 소프트 드롭, 하드 드롭
- SRS 회전(킥 테이블 포함)
- 라인 클리어 판정 및 점수 계산
- 낙하 속도 = 레벨에 따라 증가, 라인 클리어 시 레벨업
- 게임 오버 판정 및 재시작
- 기본 인게임 UI(점수/레벨/라인, 넥스트 큐 1개)
- 키보드 입력 처리

> 근거: 위 항목이 빠지면 "테트리스"라고 부를 수 없는, 게임의 정체성을 이루는 필수 요소.

### P1 — "표준 테트리스 경험 완성"
- 넥스트 큐 5개 표시
- 홀드 기능
- 고스트 피스
- 7-bag 랜덤 생성기(P0에서는 완전 랜덤으로 임시 구현 후 P1에서 교체 가능)
- 타이틀/일시정지/게임 오버 화면(React UI)
- 로컬스토리지 최고 점수 저장
- 사운드 효과(라인 클리어, 테트리스, 게임 오버, 레벨업)
- 탭 비활성화 시 자동 일시정지

> 근거: 게임 자체는 P0로 완성되지만, 이 항목들이 있어야 "제대로 만든 테트리스"로 인식되고 반복 플레이 동기(홀드/고스트/최고점수)가 생김.

### P2 — "완성도 및 확장"
- 모바일 터치 컨트롤(스와이프/버튼)
- 키 리매핑 설정 화면
- 배경음악, 볼륨 설정
- 시각 이펙트 고도화(테트리스 클리어 시 화면 흔들림, 콤보 표시)
- 백투백(Back-to-Back), 콤보 점수 보너스
- (확장 옵션) 온라인 리더보드, 대전 모드

> 근거: 게임의 재미를 해치지 않으면서 완성도/도달 범위를 넓히는 항목. MVP와 P1이 안정화된 후 여유가 있을 때 진행.
